2. Virtual Conduct & Screenshot Rules

Together matching rooms connect real individuals globally for real-time study, work, and focus. To maintain a safe, welcoming, and productive environment for neurodivergent individuals and students across all session types (Solo Sessions, 1:1 Live Body Doubling, and 24/7 Focus Spaces), you agree to strictly adhere to the following conduct rules:

  • Work & Study Purpose: Focus rooms exist solely for work, study, chores, and quiet task management. You must not use the matches for socializing, dating, marketing, or general non-focus purposes.
  • Decency & Attire: If camera connection is active, you must remain fully clothed, display an appropriate, respectful environment, and avoid displaying any offensive, harassing, or graphic material.
  • Respectful Interaction: In voice mode, verbal communication must remain polite and encouraging. Harassment, hate speech, bullying, swearing, or disruptive behavior is met with a zero-tolerance policy.
  • Screenshot Protection: Live partner sessions activate native OS screen protection to guard peer privacy. iOS detects screenshot attempts; Android natively blocks screenshots and listens for Android 14+ capture callbacks. Repeated screenshot attempts automatically trigger a 4-hour partner-session restriction.
  • Account Restrictions: Active account restrictions are cloud-backed and verified from Supabase before enabling or starting partner sessions.

3. Structured Safety Reports & Evidence Storage

Together provides a structured safety reporting system for community protection. Reports require selecting a typed reason: under 18, harassment, sexual content, threats, spam/scam, or other safety concern.

Submitting a report immediately exits the media room first so the partner is notified, submits the report to Supabase via constrained RPCs, and restricts the reported user.

For camera-session reports, Together captures a short proof clip that is uploaded securely to Supabase storage for human moderation review. Temporary local evidence files are deleted immediately after upload, and local file paths are never stored. Report restrictions execute via constrained database Remote Procedure Calls (RPCs).

5. Cloud Data, Row-Level Security & Account Control

For signed-in Supabase users, restrictions, blocks, reports, friend requests, accepted friends, profile history, settings, and friend visibility are cloud-backed using Supabase database tables protected by strict Row-Level Security (RLS). Local profile state does not decide restriction or friend truth.

Explicit Blocking: Choosing "Block User" creates an explicit cloud database block record. Session matchmakers fetch block records at start to permanently exclude blocked users from matching with you.

Friend & Streak Controls: Users can disable receiving friend requests and choose whether their focus streak is visible publicly.

Account Erasure: Account deletion purges all local profile data and permanently deletes the Supabase authentication user along with associated cloud data cleanup paths.
